## 3. P0 任务

### P0.1 修复 Docker Compose 应用数据库闭环

问题：

- API 容器连接 `siq_app`。
- Postgres 容器当前只创建 `siq`。
- `apps/api/scripts/create_database.py` 存在，但没有接入 compose 启动链路。

关键文件：

```text
infra/docker/docker-compose.yml
infra/env/docker.example
apps/api/scripts/create_database.py
docs/operations/local-development.md
```

推荐实现：

1. 在 `infra/docker/` 下新增 Postgres init 脚本目录，例如 `infra/docker/postgres-init/`.
2. 增加 `001_create_databases.sql` 或 shell init 脚本，至少创建：
   - `siq_app`
   - `siq_document_parser`
   - `siq_us`
   - `siq_hk`
   - `siq_jp`
   - `siq_kr`
   - `siq_eu`
3. 在 `infra/docker/docker-compose.yml` 中把 init 目录挂载到 `/docker-entrypoint-initdb.d:ro`.
4. 保持 `SIQ_APP_DATABASE_URL` 指向 `siq_app`，不要为了绕过问题改回单库。
5. 在文档中说明 init 脚本仅在首次初始化 volume 时执行，已有 volume 需要手动运行 `create_database.py` 或重建 volume。

验收命令：

```bash
docker compose -f infra/docker/docker-compose.yml --env-file infra/env/docker.example config >/tmp/siq-compose-config.yml
rg -n "siq_app|docker-entrypoint-initdb.d|siq_us|siq_hk|siq_jp|siq_kr|siq_eu" /tmp/siq-compose-config.yml infra/docker docs/operations
bash -n scripts/check_all.sh start_all.sh apps/api/start.sh apps/pdf-parser/run.sh apps/document-parser/run.sh
```

交付说明必须包含：

- 新增了哪些数据库。
- 对已有 volume 的迁移注意事项。
- 是否实际启动过 compose；如果没有启动，说明原因。
