# 通用文档解析回归集

## 目录定位

`eval_datasets/document_parser_cases` 保存通用文档解析链路的回归样本，用于验证不同文档类型在 artifact 合同、source map、表格关系和抽取能力上的稳定性。

## 主要内容

- `files/`：真实样本文件或本地替换样本入口
- `*.json` case 文件：描述输入、期望产物和最低质量门槛

## 与其他数据目录的边界

- 它属于 `eval_datasets/` 的回归样本子集，不是新的通用 fixture 首选目录。
- 新的小型稳定样本若要长期维护，优先放到 `datasets/fixtures` 或 `datasets/samples`。
- 单次解析运行输出不回写到这里，而应进入 `artifacts/` 或对应运行态目录。

## 可提交与不可提交内容

可提交：

- 经授权、可公开、可复现的样本说明和小型 case 定义

不可提交：

- 敏感合同全文
- 未授权研报
- 本地调试输出和解析缓存

## 运行或使用建议

- 若仓内不放真实样本，可在本地用同名文件替换 `files/` 下内容执行回归。
- 关注的不只是“能否解析完成”，还包括 `document_full`、`source_map`、`table_relations` 和抽取结果是否稳定。
- 变更 provider 或 artifact 合同时，应优先用这里的 case 做最小回归验证。

## 维护原则

- case 文件描述应足够清楚，让后来者知道输入、预期和质量门槛。
- 样本尽量控制在必要最小规模，避免目录逐渐退化成大文件仓库。
- 回归集服务稳定性验证，不应承担运行态缓存职责。
