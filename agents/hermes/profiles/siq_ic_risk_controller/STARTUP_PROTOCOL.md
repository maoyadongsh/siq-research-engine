# STARTUP_PROTOCOL.md - IC_Risk_Controller 启动检索协议（固化版 v1.0）

> **生效日期**：2026-04-30
> **协议类型**：强制启动协议
> **适用范围**：每次接收任务前的自动执行流程

---

## 一、协议目的

确保风控专家在发表观点前，已完成：
1. **项目底稿熟悉**（siq_deal_shared）
2. **私有知识库深度学习**（siq_ic_risk_controller）
3. **证据驱动的风险分析**（非直觉判断）

---


## 三、检索服务配置（由 Deal OS 后端控制）

Hermes profile 不直接连接本地 Milvus、embedding 或 rerank 服务。共享底稿、私有知识库、向量检索和重排均由 Deal OS startup-retrieval API 根据后端配置、显式 opt-in、权限和审计规则执行。

逻辑来源：
- 共享底稿：`siq_deal_shared` / deal evidence package
- 私有知识：`siq_ic_risk_controller`
- 可选增强：vector adapter / rerank adapter（响应中必须有状态与 reason）

### 5.2 评分纪律（固化）

| 条件 | 评分限制 |
|------|---------|
| 置信度 Low | 分数浮动 ±15 |
| 存在红旗项 | 评分不得超过 60 |
| 未验证假设 >3 项 | 评分不得超过 70 |
| 数据来源未标注 | 退回补正 |

## 六、自动化入口（SIQ/Hermes）

在 Hermes 中不再直接执行 OpenClaw 本地 `startup_retrieval.py`。风控专家使用 Deal OS 后端：

```text
POST /api/deals/{deal_id}/agents/siq_ic_risk_controller/startup-retrieval
```

请求体：

```json
{
  "round_name": "R1",
  "query": "{company_name} 风险 供应链 舆情 行业周期",
  "limit": 20
}
```

若 API 不可用，降级读取 `data/wiki/deals/{deal_id}` 项目包，并在报告中标注 `retrieval_degraded`。

- [ ] 红旗项已标注（如有）
- [ ] 监控指标和触发阈值已明确
