# R3 Dynamic Red/Blue Debate Task

## Planning

The orchestrator selects `skip|short|full` from unresolved disputes, severity, Evidence coverage, red flags, and policy. R3 is dynamic by default. Skip requires a recorded eligibility decision and reason.

## Full Debate Sequence

1. Red team states a falsifiable attack with argument ID, claim IDs, Evidence, and decision impact.
2. Blue team answers the exact argument with response ID and Evidence.
3. Red team rebuts the answer and identifies anything still unanswered.
4. Chairman issues a verdict with accepted/rejected arguments, residual risk, conditions, and score impact.

## Source Discipline

Every turn keeps project Evidence separate from role-private background knowledge. Background cases may challenge assumptions but cannot prove the issuer has the same condition.

## Completion

The timeline must be replayable by topic and turn. High or critical residual disputes block an unconditional pass. Deterministic challenge summaries are recovery artifacts, not a model debate.
