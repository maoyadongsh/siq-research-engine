# Models And Output

## 经典模型

按需使用并说明缺失字段。模型要嵌入对应章节，服务判断，不得堆成模型清单：
- 杜邦分析：净利率、总资产周转率、权益乘数、ROE 传导。
- 盈利质量：归母/扣非、毛利率、费用率、减值、投资收益、非经常损益。
- 现金流质量与自由现金流：OCF、资本开支、FCF、现金净增加。
- 营运资金与现金转换周期：DSO、DIO、DPO、CCC。
- 偿债安全：资产负债率、有息债务、短债覆盖、流动/速动比率、利息保障倍数。
- Altman Z-Score：字段不足时明确“不可靠计算”，不得伪造。
- 异常与舞弊风险提示：收入现金流背离、应收/存货异常、减值集中、关联交易、审计意见。
- 情景推演：改善、基准、承压三类情景，列出可证伪指标。

## 模型落点

- `profitability_and_cost`：杜邦分析、利润桥、毛利率/费用/减值/非经常性损益拆解。
- `asset_quality_working_capital`：存货周转、应收周转、DSO/DIO/DPO/CCC、资产减值风险。
- `debt_liquidity`：短债覆盖、现金覆盖、利息保障、Altman Z-Score 适用性。
- `cash_flow_quality`：经营现金流/净利润、经营现金流/收入、自由现金流、现金流来源质量。
- `industry_competition`：同业分位、成本曲线、价格传导、产品结构与竞争位置。
- `valuation_expectation_gap`：P/E、P/B、P/S、EV/EBITDA、估值分位和预期差；缺市场数据时只讨论基本面锚和数据缺口。
- `risk_chain_scenario`：因果链、情景推演、反证条件和风险缓释条件。

## 定性分析模型

- 商业模式质量：收入来源、客户/渠道、产品结构、价格权、供应链位置。
- 战略兑现模型：战略表述 -> 经营动作 -> 报表变量 -> 二级市场含义。
- 治理风险模型：事实披露 -> 信息质量 -> 融资能力/估值折价 -> 复核事项。
- 周期与竞争模型：行业周期、价格战、技术替代、份额变化、成本曲线。
- 可证伪假设：每个重要判断都要写清楚哪些后续数据会确认或推翻。

## 分析师写作模型

可见章节不是底稿字段的转写。每个 `narrative_block.items` 应写成可直接阅读的研究段落，推荐 90-180 字，至少包含“事实、解释、影响、结论/验证”四层中的三层。

章节开头的 `本节综合解读` 必须用 2-3 条自然段回答：

- 本节结论：公司在该维度处于改善、承压、脆弱、失衡、尚待验证还是证据不足降级。
- 证据基础：哪些本地年报/metrics/模型/外部补证支持这个判断，哪些关键字段缺失。
- 财务含义与验证边界：该判断如何影响盈利质量、现金流、偿债、资产质量、估值预期或风险链，后续什么信号会确认或推翻。

禁止把 `本节综合解读` 写成“研究包补充判断为、对应本地事实锚点、模型校验口径、外部搜索仅作为补证”这类流程描述。总编必须把它们改写为读者视角的结论。

正文推荐表达：

```text
公司该维度的核心矛盾不是单一指标高低，而是 XX 与 XX 的匹配度。报告期内，XX 指标为 XX，同比 XX，证据来自 XX；这一变化更可能由 XX 驱动，而非简单的规模扩张。三表联动看，XX 对利润形成支持/削弱，但 XX 尚未同步改善，因此结论应降级为观察项。后续重点看 XX 是否连续两期改善/恶化。
```

不合格表达：

```text
营业收入增长，利润改善，公司经营向好。研究包显示证据完整，外部搜索作为补充。
```

## 证据与智慧的边界

- 财务数字、同比、比率、同业分位和年报原文必须尽量引用已有 evidence，并保留 task_id、pdf_page、table_index、md_line 或明确缺口。
- 大模型的价值在于把证据连成解释：三表钩稽、成因拆解、传导路径、场景判断、反证信号和跟踪优先级。
- 没有证据的内容只能写为“推论、假设、待验证”，不得写成事实。

## 二级市场适配

- 不输出综合评分、目标价、买卖评级、投决、轮次、融资条款。
- 估值章节只讨论 P/E、P/B、P/S、EV/EBITDA、分位和预期差；缺实时市场数据时明确说明。
- 结论必须可跟踪：改善信号、恶化信号、需补充验证数据。
- A 股风险扫描必须进入治理、估值、风险情景和跟踪清单章节：ST/退市风险、非标审计意见、持续经营不确定性、交易所问询函、证监会/交易所处罚、股权质押、减持/解禁、再融资、关联方资金占用、违规担保、商誉/资产减值、政府补助依赖和主题概念与基本面脱节。未取得证据时写入 `missing_fields` 或第十四章复核清单。
- 对政策、技术主题、产业链景气和外部搜索信息，必须回答“是否已经被收入、毛利率、扣非利润、订单/客户、产能利用、存货、应收、经营现金流或同业分位验证”。未经验证只能写成外部背景或待验证假设。

## 金额与精度

- 金额统一使用亿元，保留 2 位小数；外币金额保留原币“亿欧元/亿美元/亿港元”等口径，有汇率、日期和来源时可同时给出人民币“亿元”。
- 人均、每股、每户等派生指标不是金额总量，不得归一为“亿元/人”；应展示为“元/人、万元/人、欧元/人、万欧元/人”等。
- 人均、每股、同比、增长率、占比、CAGR、外币折人民币和金额单位归一等衍生计算，必须调用 `/home/maoyd/siq-research-engine/data/hermes/home/profiles/shared/scripts/financial_calculator.py`。完整规则见 `/home/maoyd/siq-research-engine/data/hermes/home/profiles/shared/rules/financial_calculation_contract.md`。
- 比率统一使用百分比，保留 2 位小数。
- 年份使用公历年度。
- 口径必须区分合并/母公司、期末/平均、账面值/公允值。

## HTML 规则

- HTML 必须包含结构化 14 章，尽量用精美的图表展示关键数据与趋势。
  - 适合可视化的章节应优先配图表，包括但不限于：经营成果趋势、盈利能力拆解（杜邦瀑布图）、资产结构、现金流三表对比、现金转换周期分解、偿债能力指标、行业竞争格局等。
  - 图表应清晰、美观、有数据标签，避免简陋的默认样式。
- 收支拆解、利润桥、瀑布图和 Sankey/ribbon 图必须遵循本地金融图表技能规范：
  `/home/maoyd/.agents/skills/finsight-chart-craft/SKILL.md`。
  - 渲染前必须先做三表勾稽，不得先美化后补口径。
  - `营业成本` 与 `营业总成本` 必须按原始合并利润表项目名区分；若使用 `营业总成本`，不得再次扣除成本、税金、销售、管理、研发和财务费用。
  - `资产减值损失`、`信用减值损失`、`公允价值变动收益`、`资产处置收益` 必须按利润表原始符号进入桥，不得隐藏在“其他/口径差”中。
  - 收入到成本/毛利的核心拆解必须使用可交互 HTML/SVG 或 ECharts，不得降级为静态图片；关键流量宽度要与金额比例一致。
- 图表分析与视觉设计必须遵循 `rules/chart_design.md`。复杂图表应优先让 `chart_visual_designer` 输出图表蓝图或审阅意见，再由 renderer 确定性生成。
- 图表不得使用缺失值补零或估算值伪装真实数据；缺失项必须标注为未匹配、隐藏该项或进入复核队列。
- 同一图中的同类元素必须形态一致，例如收入流统一为 closed ribbon，不能出现头部大流为粗弯管、小流为细线的混搭。
- HTML 不使用侧边目录导航栏；报告主体优先保证正文、表格和图表的横向空间。
- 不得把报告工具说明、流程说明、模板说明写进正文。
- 不得把证据元数据塞进经营/盈利/现金流指标表；证据元数据只放在“数据质量与溯源声明”或引用来源。
