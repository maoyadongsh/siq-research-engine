# 章节诊断卡片模板

每个章节开头必须包含诊断卡片，提供3秒快速理解的核心结论。

## 标准格式

```markdown
┌─────────────────────────────────────────────────┐
│ 📊 章节诊断卡 - {章节名称}                       │
├─────────────────────────────────────────────────┤
│ 核心结论: {一句话总结}                           │
│ 健康度: {🟢健康/🟡预警/🔴风险}                   │
│ 关键指标: {最重要的1-2个指标及变化}              │
│ 改善信号: {什么情况下会变好}                     │
│ 证据支持: {🟢完整/🟡部分/🔴推断}                 │
└─────────────────────────────────────────────────┘
```

## 健康度评级标准

### 🟢 安全/健康
- 指标优于行业平均或历史水平
- 无重大风险信号
- 证据充分

### 🟡 预警/观察
- 指标低于行业平均但仍在合理范围
- 存在潜在风险但有缓冲
- 需要持续监控

### 🔴 风险/异常
- 指标显著劣于行业或历史
- 存在明显恶化趋势
- 可能触发连锁反应

### ➡️ 中性/稳定
- 指标与行业平均持平
- 无明显改善或恶化
- 处于常态波动范围

## 证据支持评级

### 🟢 完整证据
- 有PDF页码
- 有表格索引
- 有数据库记录
- 可完整追溯

### 🟡 部分证据
- 仅有Wiki记录
- 或仅有数据库记录
- 缺少PDF原文页码
- 部分可追溯

### 🔴 推断性结论
- 基于行业经验
- 基于历史规律
- 基于管理层表述
- 证据不足，需验证

## 使用建议

1. **放在章节最开头**：读者打开章节首先看到
2. **控制在5-7行**：超过就失去"快速"特性
3. **数字要具体**：不要只说"下降"，要说"下降30%"
4. **结论要明确**：不要模棱两可
5. **信号要可操作**：改善信号要有具体阈值

## 演示时的使用技巧

- 快速翻页时，只看诊断卡就能了解全貌
- 用颜色快速识别问题章节（🔴优先关注）
- 证据支持度体现报告可信度
- 改善信号可作为后续跟踪依据
