# Legal Opinion Quality Gate

正式法律意见书（HTML/Markdown）只有在以下条件全部满足时才允许保存到 `wiki/companies/<id>/legal/`：

## 必须失败的情况

- 缺少 `## 引用来源` 章节或章节为空。
- 引用条目少于 3 条（除非问题极其简单且自检写明）。
- 任意引用条目缺少 `source` / `source_path` / `chunk_index` 中的任意字段。
- HTML 包含暗色背景（`#0d`、`#1a`、`#000`、深紫/深蓝大面积背景）。
- 出现"已确定属于违法/犯罪"等越界定性而无监管文件支撑。
- 缺少免责声明（"不替代执业律师判断"）。
- 缺少风险提示部分。
- 出现明显的占位符 `{xxx}`、`TODO`、`待补充`。
- 出现模板残留、口语化保证或绝对承诺，例如"放心"、"完全没问题"、"一定合规"、"不会被处罚"。
- HTML `<section>` 标签数量不平衡。

## 警告项（不阻塞但必须披露）

- 引用条目少于 5 条。
- 部分引用未包含 `quote` 原文摘录。
- 某条法规仅在标题级出现，正文未引用其内容。
- HTML 文件大小超过 500KB（暗示嵌入了过多冗余内容）。
- 缺少"基于现有事实/初步/倾向/待核实/检索局限"等审慎表达。
- 缺少可执行建议，如补充核实、披露、整改、提交律师复核或后续跟踪。

## 推荐章节结构

正式意见书建议包含：

1. 标题页（公司、事项、出具日期）
2. 摘要（一段话结论）
3. 事实背景
4. 适用法规与检索路径
5. 法律分析
6. 风险提示
7. 结论与建议
8. 引用来源
9. 免责声明

每节标题为 `<h2>`，便于校验与导出 PDF。

## 写作口吻

意见书应像公司法务或律师写给管理层的工作底稿：结论克制、依据清楚、建议可执行。允许使用表格呈现风险矩阵，但关键判断必须用自然段说明事实前提、法律逻辑、证据缺口和业务后果，不应只堆字段。
