## 十四、数据质量与全链路可审计证据链

### 什么是全链路可审计证据链？

SIQ建立了从**原始PDF → 数据提取 → 数据库存储 → 分析计算 → 报告结论**的完整证据链，确保报告中的每一个数字、每一个结论都能追溯到其原始来源。

#### 证据链路径图

```
原始年报PDF
    ↓ [PDF解析引擎]
结构化数据（Markdown + JSON）
    ↓ [语义提取]
Wiki知识库 + PostgreSQL数据库
    ↓ [证据索引]
Evidence证据片段（含页码、表格位置）
    ↓ [分析计算]
财务指标 + 分析结论
    ↓ [溯源标注]
✅ 本报告（每个数字可追溯）
```

### 3.1 三表钩稽验证

本报告已对三大报表进行自动钩稽验证：

| 验证项 | 公式 | 本公司数据 | 验证结果 |
|--------|------|-----------|---------|
| 资产负债平衡 | 资产 = 负债 + 所有者权益 | 45,600 = 22,200 + 23,400 | ✅ 通过 |
| 现金流量平衡 | 期末现金 = 期初现金 + 现金流量净增加额 | 350 = 320 + 30 | ✅ 通过 |
| 利润表与现金流联动 | 净利润 + 调整项 ≈ 经营现金流 | 125 + (-40) = 85 | ✅ 合理 |

⚠️ **发现的差异**：
- 经营现金流与净利润差异40亿，主要来自应收账款增加35亿、存货增加12亿

2. 存货周转天数从45天增至65天（同业平均50天）
3. 其他应收款大幅增加（+120%），主要为关联方往来

这些异常不影响报告核心结论，但建议进一步核查。
